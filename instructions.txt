TO run namenode :
  In Git Bash: ( Works too persistent DB )
    ->cd storec
    -> chmod +x init.sh
    ->scripts/init.sh ( this will re create the DB if u r doing for first time and then run the namenode server )
    
    To use persistent DB and want to see the DB after shutting off namenode: ( use this )
    -> cd namendoe
    -> docker-compose up -d  ( this will run DB in docker)
    -> cd..
    -> python -m namenode.app.main ( starts namenode server )

To run docker-compose.yaml( to run datanode in docker container ):

  -> cd storec
  -> docker build -t storec-datanode -f datanode/Dockerfile . ( make sure in datanode/.env u provide ur datanode ip instead of 0.0.0.0)
  -> docker run --name datanode1 --env-file datanode/.env -p 50052:50052 -v storec_dn1_data:/data storec-datanode

  docker logs -f datanode-v3   # to see the logs in terminal instead of seeing in docekr desktop
  docker compose down    # To stop or remove the container 
  docker compose restart datanode  # To restart the DataNode container
